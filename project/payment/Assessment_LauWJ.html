Methodology

To develop an effective prediction model, we employed a machine learning-based approach that involved:

  • *Feature Engineering:* Raw student data was transformed into meaningful variables to capture key academic and behavioral patterns.

  • *Model Selection:* We implemented multiple machine learning models, including XGBoost, Random Forest, and Logistic Regression.

  • *Stacking Ensemble Learning:* To enhance predictive performance, we combined multiple models into an ensemble framework, leveraging the strengths of different algorithms.

  • *Hyperparameter Tuning & Class Imbalance Handling:* Optimized model performance by fine-tuning parameters and addressing data imbalance issues.

  • *Explainable AI (XAI):* SHAP (SHapley Additive exPlanations) was used to interpret model predictions, ensuring transparency and allowing educators to understand which factors most influence academic success.

This methodological approach was chosen because machine learning models outperform traditional statistical methods in handling complex, high-dimensional educational data. The integration of explainable AI also made the predictions interpretable and actionable for stakeholders in the education sector.

Conclusion & Key Findings The study successfully developed a scalable and highly accurate prediction model for CGPA forecasting. The results demonstrated that:

  • Psychological and behavioral factors (e.g., study habits, engagement levels) were just as significant as demographic factors in predicting academic outcomes.
  • Machine learning models, particularly ensemble-based approaches, significantly improved prediction accuracy compared to standalone models.
  • SHAP-based explainability enabled institutions to identify at-risk students early and tailor interventions accordingly.

This project highlights the transformative potential of AI in education, demonstrating how predictive analytics can support data-driven decision-making to foster academic success. The scalable prediction pipeline developed in this research offers educational institutions a valuable tool for proactively improving student outcomes.

B3.2. Anomaly Detection Algorithms

The anomaly detection process was applied to transaction volume and value for different payment instruments, using nine different anomaly detection methods including Z-Score, IQR, STL-Loess, Isolation Forest, LOF, DBSCAN, K-Means, Gaussian Mixture, and One-Class SVM. Each method was designed to capture different types of anomalies, ranging from statistical outliers to density-based and trend-based anomalies. After analyzing the results, DBSCAN consistently detected the highest number of anomalies across all payment instruments, followed by One-Class SVM, Gaussian Mixture, and K-Means, while Z-Score, IQR, and STL-Loess detected very few or no anomalies. This suggests that the majority of anomalies are density-based rather than purely statistical deviations.

For Charge Card transactions, DBSCAN identified 73 anomalies, while Z-Score detected only 1 in volume and none in value, suggesting that statistical deviations are minimal, but local density anomalies are significant. Credit Card transactions showed a similar pattern, with DBSCAN, One-Class SVM, and Gaussian Mixture detecting the highest number of anomalies, while STL-Loess again failed to flag any outliers. Debit Card and E-Money transactions followed the same trend, with DBSCAN consistently detecting the most anomalies, while methods like IQR and STL-Loess remained ineffective. The lack of flagged anomalies in STL-Loess suggests that most trend shifts were gradual rather than abrupt, making STL decomposition less sensitive to these variations.

Overall, density-based methods (DBSCAN, LOF, and Gaussian Mixture) appear to be the most effective in detecting anomalies in transaction data, as financial transactions often exhibit clustered behaviors rather than purely statistical outliers. Z-Score and IQR struggled to detect meaningful deviations, indicating that anomalies are not merely extreme values but involve complex distribution changes. Additionally, One-Class SVM performed well, highlighting transactions that deviated significantly from the learned normal behavior. These findings suggest that for financial anomaly detection, a combination of DBSCAN and probabilistic models like Gaussian Mixture may provide the most robust results.

# Define anomaly detection methods
anomaly_methods = ['ZScore', 'IQR', 'STL_Loess', 'IsoForest', 'LOF', 'DBSCAN','KMeans','GaussianMixture', 'OneClassSVM']

# Function to detect anomalies
def detect_anomalies(df, column):
    results = {method: np.zeros(len(df)) for method in anomaly_methods}

    # Z-Score method
    z_scores = np.abs(zscore(df[column]))
    results['ZScore'] = np.where(z_scores > 1.96, -1, 1)

    # IQR method
    Q1 = df[column].quantile(0.25)
    Q3 = df[column].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR
    results['IQR'] = np.where((df[column] < lower_bound) | (df[column] > upper_bound), -1, 1)
        
    # Isolation Forest
    iso_forest = IsolationForest(contamination=0.2, random_state=42)
    results['IsoForest'] = iso_forest.fit_predict(df[[column]])

    # Local Outlier Factor
    lof = LocalOutlierFactor(n_neighbors=5, contamination=0.2)
    results['LOF'] = lof.fit_predict(df[[column]])

    # DBSCAN
    dbscan = DBSCAN(eps=0.5, min_samples=100)
    results['DBSCAN'] = np.where(dbscan.fit_predict(df[[column]]) == -1, -1, 1)

    # KMeans
    kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
    clusters = kmeans.fit_predict(df[[column]])
    results['KMeans'] = np.where(clusters == np.argmax(np.bincount(clusters)), 1, -1)

    # # Gaussian Mixture Model
    gmm = GaussianMixture(n_components=3, random_state=42)
    gmm.fit(df[[column]])
    gmm_probs = gmm.predict_proba(df[[column]])
    results['GaussianMixture'] = np.where(gmm_probs[:, 0] < 0.05, -1, 1)

    # # One-Class SVM
    oc_svm = OneClassSVM(nu=0.95, kernel="rbf", gamma=0.1)
    results['OneClassSVM'] = oc_svm.fit_predict(df[[column]])

    # Combine results
    df_results = pd.DataFrame(results)
    df_results["Anomaly Count"] = (df_results == -1).sum(axis=1)
    df_results["Final Anomaly"] = np.where(df_results["Anomaly Count"] >= 4 , -1, 1)
  
    return df_results["Final Anomaly"], df_results

# Apply anomaly detection for each instrument type on both Volume and Value
anomaly_results = {}
all_anomaly_counts = {}

for instrument in df_instrument_trend["Instrument Type"].unique():
    subset = df_instrument_trend[df_instrument_trend["Instrument Type"] == instrument].copy()

    # Detect anomalies for Volume
    subset["Volume Anomaly"], volume_results = detect_anomalies(subset, "Volume (Million)")

    # Detect anomalies for Value
    subset["Value Anomaly"], value_results = detect_anomalies(subset, "Value (RM Million)")

    anomaly_results[instrument] = subset

    # Store anomaly counts for both Volume and Value
    volume_anomalies = volume_results.iloc[:, :-2].apply(lambda x: (x == -1).sum())
    value_anomalies = value_results.iloc[:, :-2].apply(lambda x: (x == -1).sum())
    all_anomaly_counts[instrument] = {
        "Volume Anomalies": volume_anomalies,
        "Value Anomalies": value_anomalies
    }

# Combine all results
final_anomaly_data = pd.concat(anomaly_results.values())

# Print consolidated anomaly counts at the end
print("Anomaly Counts Per Method Across All Instrument Types:\n")
for instrument, anomaly_data in all_anomaly_counts.items():
    print(f"Instrument Type: {instrument}")
    print("Volume Anomalies:")
    print(anomaly_data["Volume Anomalies"])
    print("Value Anomalies:")
    print(anomaly_data["Value Anomalies"])
    print("-" * 50)
